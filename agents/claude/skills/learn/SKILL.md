このセッション（会話・調査等）を振り返り、CLAUDE.md / rules / skills / sub-agents の新規作成・更新を提案する。

## 保存先

### グローバル vs プロジェクト固有

- グローバル（`~/.dotfiles/agents/claude/`）: 全プロジェクト・全セッションに適用したい知識
- プロジェクト固有（作業リポジトリの `.claude/`）: 特定のリポジトリ・チームにのみ適用したい知識

### 種別

以下の質問で上から順に判断する。最初に YES になった種別に保存する。

1. 毎セッション自動的に必要か？常に守ってほしいルール・制約か？
   → YES: **CLAUDE.md**（グローバル: `~/.dotfiles/agents/AGENTS.md`）

2. 特定のファイル種別・ディレクトリを操作するときだけ適用したいか？
   → YES: **rules**（`paths` glob で絞る）

3. 必要なときだけ呼び出す手順・ワークフロー・参照資料か？
   → YES: **skills**

4. 以下のいずれかに該当するか？
   - メインの会話コンテキストを汚したくない（大量ファイル読み込みなど）
   - 複数ターンにわたって状態を保持する必要がある
   - ツール制限・独自権限モードが必要
   - 会話をまたいで記憶を蓄積させたい（`memory`）
     → YES: **sub-agents**

## 手順

1. セッション（会話・調査等）で得た知識を以下のテーブル形式で提示する（判断はしない）

| #   | 知識 | 操作 | 対象 |
| --- | ---- | ---- | ---- |

- 操作: CREATE or UPDATE
- 対象: CLAUDE.md / rules / skills / sub-agents のいずれかの候補（ファイルパス）

2. ユーザーが承認したものだけ実行する
